# 文献数据库配置指南

> 在使用本 skill 之前，请按本文件的格式配置你自己的文献库。

---

## 第一步：填写你的数据库根路径

将下面这行替换为你实际存放文献库的目录：

```
数据库根路径：{YOUR_DATABASE_PATH}
```

示例：
```
数据库根路径：D:/my-research/literature-database/
```

---

## 第二步：建立你的文献库文件

每个文件对应一个研究主题，Claude 在验证假设时会按需读取。

### 文件格式要求

每个文献库文件应包含以下结构：

```markdown
# [主题名称]文献库

## 核心理论框架
[简述该领域的2-3个主流理论及其核心主张]

## 精读文献条目

### Author, A. (Year). Title. *Journal*, Vol(Issue), pp.
- **核心主张**：[该论文的中心论点，1-2句]
- **对竞争性假设的贡献**：[该论文支持哪个方向的预测，为什么]
- **被引量**：[TC: xxx]（可选，但有助于优先级排序）

### Author, B. (Year). Title. *Journal*, Vol(Issue), pp.
...
```

### 推荐的文件组织方式

| 文件名 | 建议内容 |
|--------|---------|
| `00_索引.md` | 所有文件的主题速览与议题→文件速查表 |
| `01_[核心理论].md` | 你研究领域最重要的理论框架文献（精读，直接可读） |
| `02_[子主题].md` | 特定子议题的文献（精读，直接可读） |
| `03_[大型索引].md` | 大批量检索结果索引（**仅用 Grep 搜索，不整体读**） |

**文件大小建议**：
- < 200行：Claude 可以直接整体读取
- > 200行：设计为只用关键词 Grep，不整体读

---

## 第三步：填写议题→文件速查表

将下表替换为你自己的文献库内容，这是 skill 运行时的导航核心：

| 如果议题涉及… | 优先读哪个文件 | 操作方式 |
|-------------|--------------|---------|
| [你的研究主题A] | `01_xxx.md` | 直接读 |
| [你的研究主题B] | `02_xxx.md` | 直接读 |
| [你的大型索引] | `03_xxx.md` | Grep: `关键词` |

---

## 第四步：列出高引经典的理论张力（可选但推荐）

这个表格让 skill 能快速定位"谁和谁对立"，是竞争性假设的理论锚点：

| 理论张力 | 一端（文献支撑） | 另一端（文献支撑） |
|---------|--------------|----------------|
| [你领域的核心争论1] | [支持正效应的代表文献] | [支持负效应的代表文献] |
| [你领域的核心争论2] | ... | ... |

---

## 使用示例

配置完成后，当 skill 运行时会：
1. 读取本文件，获取数据库路径和议题映射
2. 根据用户输入的研究议题，定位相关文件
3. 读取或 Grep 搜索相关文献
4. 基于真实文献构建竞争性假设的理论支撑

**重要**：skill 只会引用数据库中真实存在的文献，不会凭空生成引用。数据库越充实，验证结果越可靠。
