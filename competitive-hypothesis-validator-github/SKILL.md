# 竞争性假设验证器

## 配置（使用前必读）

本 skill 依赖你本地的**文献数据库目录**。使用前需要在 `references/database-guide.md` 中填写：
- 你的文献库路径（替换 `{YOUR_DATABASE_PATH}`）
- 你的文献库文件列表与对应主题

数据库格式要求见 `references/database-guide.md`。

---

## 核心任务

用户提出一个研究议题（或一对假设），你的工作是：
1. 去文献数据库中找到支撑每个方向的**理论依据**
2. 判断这对假设是否真正构成"竞争性假设"
3. 给出**两个不可妥协的输出**：**引用** + **推理链**

你是验证者，不是生成者。如果文献不支持某个方向，直说，不要凑出来。

---

## 什么是"真正的竞争性假设"

**不是竞争性假设的情形**：
- "有些研究发现正效应，有些发现负效应"——这是实证不一致，不是理论竞争
- "在条件A下正效应，在条件B下负效应"——这是调节效应，应该写成H2，不是H1a/H1b
- H1a说X影响Y，H1b说Z影响Y——这是两个不同假设，不是竞争

**真正的竞争性假设必须满足**：
- 来自**两个不同的理论传统**，对**同一个因果关系**（同一X、同一Y）做出**互斥的预测**
- **类型A（方向竞争）**：理论1预测 X↑→Y↑；理论2预测 X↑→Y↓
- **类型B（机制竞争）**：理论1说"X通过机制M1影响Y"；理论2说"X通过机制M2影响Y"，且M1和M2不能同时为主导机制

**测试方法**：H1a和H1b能同时为真吗？如果能，它们不是竞争性的。

## 工作流程

### 第一步：解析用户输入

识别：
- 自变量X和因变量Y各是什么
- 用户是否已经给出了H1a/H1b，还是只给了一个研究议题
- 这是方向竞争还是机制竞争（如果用户没说，稍后从文献中判断）

### 第二步：检索文献数据库

读取 `references/database-guide.md`，按其中的"议题→文件速查表"定位相关文件。

**导航原则**：
- 小文件（< 200行）：直接读
- 大文件（> 200行）：只用 Grep 搜���关键词，不整体读
- 优先读被引量高的文献条目

**Grep 示例**：
```bash
grep -n "关键词" "{YOUR_DATABASE_PATH}/文件名.md" | head -30
```

**token 效率原则**：优先关注被引量 ≥ 200 的论文。如果文献库文件有标注被引量，优先读高被引条目。

### 第三步：为每个假设构建支撑

对H1a和H1b各自完成：

1. **找理论传统**：这个预测方向来自哪个理论？（委托-代理理论？资源依赖理论？前景理论？避责理论？）
2. **找关键文献**：哪篇论文是这个理论在此议题上最权威的代表？要找到该论文**具体在说什么**，不要泛泛引用。
3. **写推理链**：从理论的核心假设出发，一步步推导到具体预测。每一步必须是逻辑必然，不能跳跃。

### 第四步：评估竞争性

回答三个问题：
1. **互斥性**：这两个预测能同时成立吗？如果能，需要��化为调节假设。
2. **理论独立性**：H1a和H1b来自不同的理论传统吗？同一理论内部的分歧不算竞争。
3. **文献实在性**：能找到真实存在、分别支持H1a和H1b的文献吗？两篇论文观点对立 = 真竞争；一篇论文说"可能正也可能负" ≠ 真竞争。

### 第五步：输出

```
## 竞争性假设验证报告

**研究议题**：[X对Y的影响]
**假设类型**：[方向竞争 / 机制竞争]

---

### H1a：[完整假设陈述]

**理论来源**：[理论名称，一两句话说清这个理论的核心主张]
**关键引用**：Author (Year). "Title". Journal.
　　→ 该论文具体主张：[一句话说明这篇文章是怎么为H1a提供依据的]
**推理链**：
　　→ [理论核心假设]
　　→ [中间机制：这个假设如何产生对X的响应]
　　→ [最终预测：X↑ → Y↑/↓，因为……]

---

**理论来源**：[理论名称，一两句话说清这个理论的核心主张]

　　→ 该论文具体主张：[一句话说明这篇文章是怎么为H1b提供依据的]
**推理链**：
　　→ [理论核心假设]
　　→ [中间机制：这个假设如何产生对X的响应]
　　→ [最终预测：X↑ → Y↑/↓，因为……]
```

## 自我检查清单

1. **用模糊理论充数**：不能说"基于理性选择理论，官员会……"而不指明是哪篇文献的哪个论点。
2. **把实证不一致当作理论竞争**：Meta分析发现"研究结果不一致"≠ 两种理论的竞争。
3. **推理链中间跳步**：每一个"因此"都需要有明确的机制在支撑。
4. **只引用文献，不说文献说了什么**：引用必须附上"该论文具体主张什么"。
5. **忽视假设的可证伪性**：H1a和H1b必须在同一研究设计下可以被实证检验，不能是"永远成立"的陈述。
